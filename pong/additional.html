Required Modularity for Pong

New Required Part of Pong Assignment

The pong assignment suggests taking a modular/object approach to the paddle in the game. This suggestion has been promoted to a requirement. Here are the details.
The Original Description
The assignment suggests defining a data structure for a paddle and then says:

Having created the paddle data structure, you need to write a few simple operations for it: init_paddle(), up_paddle(), down_paddle(), and is_a_hit(y,x). The init_paddle() function must initialize the paddle data structure and draw it on the screen. The Revised, Clarified Requirement
You must implement the paddle as a separate C file, called paddle.c. That file is required to:
  • use static data to store the state of the paddle, preventing other parts of the program from having direct access to the state of the paddle
  • implement a set of functions for moving the paddle, setting paddle attributes, and reading paddle information
  • have a header file called paddle.h that contains declarations of the non-static functions in paddle.c
Questions?
If you are not sure how to create a separate abstract object or layer in C, look at the utmplib.c file from lecture 2. Or send us email with specific questions about your design.
General Modularity
Some of the regular design points on pong are based on applying this modular approach to other objects in the system. You can write ball.c and ball.h files to implement the ball.
Extra Credit
You can earn up ten points of extra credit by adding a significant addition to the game. For example, you could allow several balls in the game at once, or you could allow the paddle to move forward and backwards, or you could create a two-player game. Displaying the score or printing little messages is nice but is not a significant addition.
